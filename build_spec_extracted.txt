3. Database Schema (CO4 — schema.sql)
Generate schema.sql with the SQL below. Python reads credentials from constants at the top of database.py.
All tables use InnoDB with foreign key constraints.
users
CREATE TABLE users ( user_id INT AUTO_INCREMENT PRIMARY KEY, username VARCHAR(50) NOT
NULL UNIQUE, password VARCHAR(255) NOT NULL, -- bcrypt hash email VARCHAR(100) NOT NULL,
role ENUM("Admin","Agent","Customer") NOT NULL, avatar VARCHAR(255) DEFAULT NULL, --
path to image file (CO5) created_at DATETIME DEFAULT NOW() ) ENGINE=InnoDB;
tickets
CREATE TABLE tickets ( ticket_id INT AUTO_INCREMENT PRIMARY KEY, customer_id INT NOT
NULL, category VARCHAR(50), complaint TEXT NOT NULL, screenshot VARCHAR(255) DEFAULT
NULL, -- image path (CO5) priority ENUM("High","Medium","Low") DEFAULT "Low", status
ENUM("Open","In Progress","Closed") DEFAULT "Open", assigned_to INT DEFAULT NULL,
created_at DATETIME DEFAULT NOW(), resolved_at DATETIME DEFAULT NULL, FOREIGN KEY
(customer_id) REFERENCES users(user_id), FOREIGN KEY (assigned_to) REFERENCES
users(user_id) ) ENGINE=InnoDB;
feedback
CREATE TABLE feedback ( feedback_id INT AUTO_INCREMENT PRIMARY KEY, ticket_id INT NOT
NULL, customer_id INT NOT NULL, rating INT CHECK (rating BETWEEN 1 AND 5), comments
TEXT, given_at DATETIME DEFAULT NOW(), FOREIGN KEY (ticket_id) REFERENCES
tickets(ticket_id), FOREIGN KEY (customer_id) REFERENCES users(user_id) ) ENGINE=InnoDB;
remarks
CREATE TABLE remarks ( remark_id INT AUTO_INCREMENT PRIMARY KEY, ticket_id INT NOT NULL,
agent_id INT NOT NULL, note TEXT NOT NULL, added_at DATETIME DEFAULT NOW(), FOREIGN KEY
(ticket_id) REFERENCES tickets(ticket_id), FOREIGN KEY (agent_id) REFERENCES
users(user_id) ) ENGINE=InnoDB;
DB constants in database.py: DB_HOST = "localhost" | DB_USER = "root" | DB_PASS = "your_password" | DB_NAME
= "complaint_db"

7. Feature Implementation Details
7.1 Auto-Priority Detection (CO1 + CO2 — utils.py)
CO1: control flow with for-loop and conditional return. CO2: KEYWORDS is a dict — a required data structure.
KEYWORDS = { "urgent": "High", "broken": "High", "not working": "High", "crash": "High",
"error": "High", "fail": "High", "slow": "Medium", "delay": "Medium", "issue": "Medium",
"query": "Low", "info": "Low", "question": "Low" } def auto_priority(complaint_text):
"""Scan complaint for keywords and return priority level.""" text =
complaint_text.lower() for keyword, level in KEYWORDS.items(): if keyword in text:
return level return "Low"
7.2 SLA Breach Colour Coding (CO1 + CO2 — utils.py)
CO1: datetime arithmetic and nested conditionals. CO2: SLA_LIMITS is a dict.
SLA_LIMITS = {"High": 2, "Medium": 24, "Low": 72} # hours def check_sla_color(priority,
created_at): """Return Tkinter tag name if ticket has breached SLA, else empty
string.""" from datetime import datetime hours_old = (datetime.now() -
created_at).total_seconds() / 3600 limit = SLA_LIMITS.get(priority, 72) if hours_old >
limit: color_map = {"High": "red", "Medium": "orange", "Low": "yellow"} return
color_map.get(priority, "") return "" # In any Treeview: # tree.tag_configure("red",
background="#FADBD8") # tree.tag_configure("orange", background="#FDEBD0") #
tree.tag_configure("yellow", background="#FEFDE7")
7.3 Live Dashboard Charts (CO4 — admin_window.py Tab 1)
fig, axes = plt.subplots(1, 3, figsize=(12, 3), tight_layout=True) # axes[0] — pie:

===== PAGE 9 =====
8. Coding Rules for the Agent
Naming
snake_case functions/variables. PascalCase classes. ALL_CAPS constants. No
single-letter vars except i/j.
Function length
Max 25 lines per function. Split longer functions into helpers with descriptive names.
Docstrings
One-line docstring on every function and class. Example: """Return priority level string
for the complaint text."""
No magic numbers
SLA_LIMITS and KEYWORDS must be module-level dicts in utils.py. Never hardcode
2, 24, or 72 elsewhere.
Error handling
Wrap every DB call in try/except. Show messagebox.showerror. Never let the app crash
silently.
No global state
Pass db and current_user into each window __init__. Store as self.db and self.user.
Password hashing
bcrypt.hashpw on register, bcrypt.checkpw on login. Plain text passwords are a fail
condition.
Input validation
Check empty fields before every DB insert. Show an error Label in the form, not just a
popup.
Imports
Standard library first, then third-party (mysql, matplotlib, PIL), then local modules.
CO5 isolation
All Pillow calls go through image_utils.py only. Window files call image_utils functions,
never PIL directly.
main.py size
main.py must be under 20 lines — entry point only. One call to db.connect(), one
LoginWindow, root.mainloop().
